# 03 · "贴图 A → 密度重分布 → 贴图 B" 这件事自洽吗？

> 你的原始表述：给定纹理贴图 A，通过纹理密度重新分布贴图上的布局，得到新贴图 B，
> 使得重新计算的密度是均衡的。你觉得这是自洽的。
> 结论先行：**逻辑成立，但只有在把"密度"定义锚定在 3D 表面上时才严格自洽**；
> 并且要区分两种不同的"密度"——它们对应两个不同的操作、两个不同的价值场景。

---

## 1. 先把"密度"拆成两个概念

你的世界地图/人口密度类比（见 `/root/youjiaZhang/纹理密度/纹理密度思路.md`）其实同时揉进了两个概念，拆开之后一切都清晰了：

| 概念 | 定义 | 类比 | 业界叫法 |
|---|---|---|---|
| **几何纹理密度** (geometric texel density, TD) | 单位 3D 表面积分到多少纹素：`TD(f) = res · sqrt(A_uv(f) / A_3d(f))`，单位 px/cm 或 px/m | 地图上"每平方公里国土占多少地图面积"——**自然地理地图**追求的就是它处处一致（等积投影） | texel density，美术日常说的就是这个 |
| **内容密度** (content density) | 单位 3D 表面积上承载了多少视觉信息（高频细节、logo、文字…） | **人口密度**——中国印度信息多，撒哈拉信息少 | 学术上对应 signal-specialized / content-aware |

于是"均衡"也有两种目标：

- **目标 U（uniform TD）**：让几何 TD 处处相等 → UV 面积 ∝ 3D 面积。
  这与贴图内容完全无关，只看几何。附图里 GOOD/BAD 棋盘格对比展示的正是它：
  GOOD = 棋盘格在表面上处处等大；BAD = 有的部位格子被拉大（欠采样）、有的被压小。
  这是**业界的默认标准**（Blender 的 Average Islands Scale、各种 TD 工具做的都是它）。
- **目标 C（content-aware TD，即"人口地图"）**：让**每个纹素承载的信息量**相等 →
  UV 面积 ∝ 3D 面积 × 内容权重。细节多的区域拿更多纹素。
  这是你类比里真正想说的 cartogram（变形统计地图），也是研究上的增量所在。

你原話裡的"根据纹理的实际密度去重新布局"指向目标 C，而"重新计算的密度是均衡的"两种目标都说得通。**两者都是自洽的操作，但自洽的理由和适用场景不同**，下面分别论证。

## 2. 自洽性的严格论证

### 2.1 操作是良定义的（well-defined）

给定 mesh `(V, F)`、旧 UV `Φ_A`、贴图 A，重分布 = 求新 UV `Φ_B`，然后重采样：

```
对 B 中每个纹素 p:  B(p) = A( Φ_A( Φ_B⁻¹(p) ) )
```

即"新纹素 → 3D 表面点 → 旧 UV → 取色"。这就是标准的 texture transfer / rebake，
你的 `tdopt/rebake_texture.py` 实现的正是它。每一步都是确定性映射，操作良定义。✅

### 2.2 关键条件：密度必须是"表面附着量"，否则会循环定义

这是整个自洽性论证里最容易踩的坑，值得写清楚：

- 如果把"内容密度"定义在**贴图 B 的像素空间**里（每个 texel 有多少高频能量），
  那么重排 → rebake 会改变它（重采样带来的模糊会削掉高频），
  于是"按密度重排"之后再测密度，得到的是**另一个**密度场——迭代不收敛，定义循环。❌
- 如果把它定义为**单位 3D 表面积上的信息量**（per-face content weight，附着在表面上），
  它就是 UV 无关的不变量：无论怎么重排布局，表面上"该有多少细节"不变。
  此时"让 UV 面积 ∝ 3D 面积 × 权重"是一个普通的、目标固定的优化问题。✅

你的 tdopt 恰好就是这么做的（`content_weight.py` 按 face 计算权重）——
所以你直觉上觉得"这事是对的"，严格化之后确实是对的，**条件就是密度锚定在表面**。

### 2.3 不动点 / 幂等性（"自洽"的数学含义）

把重分布记作算子 `R`：`R(布局) = 按表面附着的密度目标重新分配 UV 面积后的布局`。

- 对目标 U：`R` 的不动点是"UV 面积 ∝ 3D 面积"的布局。对已均衡的布局再作用一次，
  什么都不变：`R(R(x)) = R(x)`，幂等。✅
- 对目标 C：同理，不动点是"UV 面积 ∝ 3D 面积 × 内容权重"的布局，幂等。✅

一个操作存在不动点、且作用一次即达到不动点，这就是"自洽"最实在的数学表述。
你的直觉可以升级成一句严格的话：

> **只要密度目标定义在 3D 表面上，"按密度重排布局"就是一个有唯一目标、
> 幂等、一步收敛的重分配操作——自洽。**

### 2.4 但有一条信息论上的"不等式"必须正视

Rebake 是重采样，**不能创造信息**：

- A 里本来就欠采样（TD 低）的区域，B 给它更多纹素后得到的是**插值放大的模糊像素**，
  分配均衡 ≠ 实际清晰度均衡；
- A 里高 TD 的区域被压缩后，细节**不可逆丢失**。

所以对"A → B 纯重排"要有清醒预期：

> **A → B 的价值是"像素预算重分配"，不是"画质提升"。**
> 信息量上 B ≤ A（Data Processing Inequality 的朴素版本）。

这不推翻自洽性——操作本身依然良定义、幂等——但它决定了这套东西**在哪个场景下才真正产生价值**：

| 场景 | 纹理来源 | 重分布的价值 |
|---|---|---|
| 资产修复（A→B rebake） | 只有贴图 A | 有限：只能"劫富济贫"，救不了本来就糊的区域。适合做**验证实验/oracle**（你已做的 Exp01–06 就是这个角色） |
| **白模 + 生成**（leader 的需求） | 纹理尚不存在，由生成器/烘焙产出 | **完整**：布局先行决定了生成器的像素预算投向哪里，没有任何重采样损失。密度控制在这里是"分配未来的预算"而非"搬运过去的像素" |

这正是 leader 需求比 A→B 实验更本质的原因：**密度控制的正确位置是在纹理生成之前的
UV 布局阶段**。你的 `whitemodel.py` 已经站在了正确的位置上。

## 3. 世界地图类比的边界：cartogram 的教训

人口 cartogram（如 Gastner–Newman 扩散法）为了让面积 ∝ 人口，会把国家形状扭曲得
面目全非——这在 UV 里恰好对应**角度失真**。而纹理映射里角度失真在感知上往往比
面积不均**更糟**（棋盘格变菱形、条纹弯曲，比格子大小不一更刺眼）。

所以类比要打一个补丁：**UV 的 cartogram 必须在 chart 粒度上做刚性缩放
（或 chart 切分），而不是连续翘曲**。这不是理论推断，你自己的实验已经给出了证据：

- `relayout.py` 头部注释明确写着优先序："chart 内部尽量保持刚性，chart-split 是推荐主方法，
  face-level ARAP 只是激进的 fallback / 内容匹配上界，绝不是默认"；
- Exp04（bounded ARAP pareto）测出的正是连续翘曲在角度失真上的代价
  （`recommended_config.json` 里 `max_anisotropy: 161` 这种数字就是警报）。

一句话：**类比成立，但"人口均衡地图"在 UV 里的正确实现是"重新划分并缩放行政区"，
而不是"把大陆捏变形"。**

## 4. 其余工程性约束（不影响自洽性，但影响可用性）

1. **打包（packing）**：面积分配目标要和 packing 效率、padding/gutter、mip 渗色一起权衡。
   分配误差 `target_err_vs_demand` 与 `packing_efficiency` 天然互相牵制（你的验收阈值
   `ACC` 字典已经在管理这组权衡）。
2. **相对 vs 绝对**：重分布只决定**相对**分配；绝对 TD（px/cm）由图集分辨率决定，
   是事后一个自由变量（`recommend_resolution()` 的 industry anchoring 思路是对的）。
3. **接缝**：重排改变不了接缝拓扑（除非 chart-split），接缝处的 TD 突变要靠 padding 兜底。

## 5. 结论

1. **你的判断正确**：按密度重分布布局、使重新计算的密度均衡，是自洽的——
   前提是把密度定义为表面附着量（你的实现已经满足）。
2. 要区分 **uniform TD（几何均衡，业界标准，内容无关）** 和
   **content-aware TD（信息均衡，"人口地图"，研究增量）**——两个目标都自洽，
   前者是后者 `内容权重 ≡ 1` 的特例（对应 `whitemodel.py` 里 `gamma = 0`）。
3. **A→B 纯 rebake 不增加信息**，它的正确角色是验证工具；
   密度控制的产品级价值在**生成前的白模布局**——这恰好就是 leader 的需求场景。
4. 类比的修正：cartogram 要做在 chart/part 粒度（刚性缩放 + 切分），
   不做连续翘曲——与你 Exp01–06 的实验结论一致。
