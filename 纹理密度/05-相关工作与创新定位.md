# 05 · 相关工作与创新定位

> 基于多路并行文献调研（学术五条线索 + AI texturing 管线 + 工具链）。
> 核心结论：**"在 part-based UV 展开之上、由（将要生成的）内容驱动的 per-part
> texel density 分配"没有任何已发表工作完整做过**——三个最近邻各缺一角，
> packing 阶段的 per-chart scale 是文献里"无人认领的自由度"。
> 但有一条竞争预警（§5）。

---

## 1. 五条学术线索

### 线索 1 · Signal-specialized parameterization（内容感知分配的鼻祖）

- **Sander, Gortler, Snyder, Hoppe. "Signal-Specialized Parametrization." EGWR 2002**：
  从 signal 重建误差的 Taylor 展开推导 signal-stretch metric，预积分成 per-triangle 的
  Integrated Metric Tensor（IMT）——signal 变化快的三角形在 UV 中自动放大。
  这就是"细节多的地方多给 texel"的奠基工作，**比我们早 24 年提出同一个直觉**。
- **Tewari et al., SGP 2004**：把重建假设改成 piecewise-linear（贴近 GPU bilinear 采样）。
- **工程化落地：Microsoft UVAtlas 的 IMT 接口**（`UVAtlasComputeIMTFromPerTexelSignal`）
  ——20 年前就能"按 normal map 细节分配 UV 面积"。
- **Balmelli et al. "Space-Optimized Texture Maps." EG 2002 (Best Paper)**：从图像侧
  warp 使频率内容空间均匀——**恰好是你"世界地图→人口地图"类比的图像域版本**，
  说明这个直觉在学术史上是被最佳论文级别认可过的。
- **缺角**：需要 signal 已存在（贴图/重建已知）；per-triangle 连续 warp（角度失真代价）；
  无 part 语义；不动 packing 的 chart scale。

### 线索 2 · Atlas packing 文献：uniform scale 是公理，也因此是空白

- Box Cutter (SIGGRAPH 2018)、**Atlas Refinement with Bounded Packing Efficiency
  (Liu et al., SIGGRAPH 2019)**、TABI (2026)、学习类 packer——全部默认
  **charts 间统一相对 scale**，只优化 packing efficiency 和边界长度；
  TABI 甚至把 "chart downscaling" 当作要消灭的缺陷。
- **含义**：把 per-chart/per-part scale 作为内容驱动的自由度引入 packing，
  在 packing 文献里是新问题（要同时管 PE、密度跳变、seam mismatch）——
  相对 Liu 2019/TABI 有明确的增量贡献点。

### 线索 3 · Importance-driven / 自适应分辨率（绕开 atlas 的旁路）

Adaptive Texture Maps (2002, 硬件 indirection)、**Ptex** (EGSR 2008, per-face 独立分辨率)、
Mesh Colors (TOG 2010)、TextureMontage (SIGGRAPH 2005)、Painting Detail (2004,
按用户绘画分配)、Feature-aware Texturing (2006, saliency warp)。
**缺角**：三种成熟路径（indirection / 抛弃 UV / 用户驱动 warp）没有一个是
"在标准 UV atlas 管线内、按语义 part、面向将要生成的内容"做分配——
而标准 UV atlas 是 DCC/引擎生态的硬约束，绕开 UV 的方案（Ptex）二十年没进游戏管线。

### 线索 4 · 神经 UV 方法：无人做 density，Nuvo 甚至反向

- **Nuvo (2023)**：L_stretch 显式把 scale 推向**均匀**，每 chart 固定分辨率——
  是唯一管 scale 的神经方法，方向与 content-aware 相反；
- FlattenAnything (NeurIPS 2024)、FlexPara (2025)、AUV-Net (CVPR 2022)：
  只有 distortion / cycle-consistency / 跨物体对齐目标，无 density 概念。

### 线索 5 · 与纹理生成的耦合：痛点已被点名，形式化无人做

来自 AI texturing 管线调研的直接证据：

- **SyncMVD** 原文："The UV mappings are by default, automatically unwrapped using
  XAtlas **to avoid potential poor UV layout**"；512² latent texture "sufficient
  for most objects **with a correct UV layout**"——坏 UV 直接损害生成质量的最直白论文表述；
- **Meta 3D TextureGen**：训练数据层面**直接过滤掉**Smart Project 展不好的物体；
  并配了 UV 空间超分网络（1K→4K）——用算力补密度，而不是把密度分对；
- **Paint3D / Point-UV Diffusion / TEXGen / UniTEX**：四篇都明文陈述 UV 碎片化
  破坏 UV 域生成/补全（UniTEX 因此干脆绕开 UV 提出 Texture Functions——
  这也说明组内 TDF 选 UniTEX 底座与本课题天然同源）；
- **Hunyuan3D-2 社区**：>100k faces 的 AI mesh 让 xatlas 失败/产坏 UV，
  社区专门维护 xatlas patch——工程痛点实锤；
- **最接近的现代工作：Knodt, Pan, Wu, Gao. "Joint UV Optimization and Texture
  Baking." TOG 43(1), 2024**：动机原文 "texels can be wasted on small or unimportant
  mesh regions, leaving salient regions ill represented"；用可微渲染联合优化 UV 与
  贴图，content-aware image-based symmetric Dirichlet energy 按局部视觉误差加权
  distortion（+3–3.5 PSNR vs NVDiffModeling）。**缺角**：面向 high-poly→low-poly
  baking（参考外观已存在）、全局连续 warp、无 part 结构、不接生成管线。
- **相邻领域回流机会**：FACT-GS (2025) 在 2DGS 上把"按频率调制采样密度"做成了
  可微问题（deformation field 的 Jacobian 调制 per-splat 密度）；A²TG (2026)
  自适应分配 per-Gaussian texture 尺寸——思想尚未回流到 mesh UV atlas。

## 2. Gap 矩阵：三个最近邻各缺一角

| | content-aware | part 结构 | 面向生成（signal 未知） | 动 packing scale | 标准 UV 管线兼容 |
|---|---|---|---|---|---|
| Signal-specialized (2002/04) + UVAtlas IMT | ✅ | ❌ | ❌ 需已有 signal | ❌ | ✅ |
| Joint UV Opt & Baking (TOG 2024) | ✅ 可微 | ❌ | ❌ baking 场景 | ❌ 全局 warp | ✅ |
| PartUV (SIGA 2025) | ❌ 全文未提 density | ✅ | ✅（为 AI mesh 设计） | ❌ 留白 | ✅ |
| **本课题（PartUV × 密度分配）** | ✅ | ✅ | ✅ 几何+语义先验 / 两遍法实测 | ✅ | ✅ |

**我们的组合创新 = 用"几何 + 语义先验 + 两遍法实测"解决"signal 未知"（这是
2002 年经典方法用不上现代生成管线的根本原因；已决策不依赖 TDF，见 06 决策记录），
用 PartUV 的 part tree 解决分配粒度，用密度感知 packing 认领那个"无人认领的
自由度"。** 可直接复用的现成组件：Sander 的 IMT（密度目标的经典定义）、
Knodt 的 content-aware symmetric Dirichlet（可微实现参考）、
FACT-GS 的 Jacobian 调制（连续重分配参考）、PartUV 的 part tree 与 τ 界。

## 3. PartUV 代码生态（已核实，2026-07）

- **已开源**：https://github.com/EricWang12/PartUV （一作个人账号；C++ 53% + Python 39%
  + CUDA；pip 包 `partuv` v0.1.2，cp39–313 manylinux，仅 Linux）。
  含完整管线：PartField 推理 → C++ 递归 unwrap（默认 ABF，τ=1.25，
  config 无任何 packing/scale/density 参数）→ 可选 packing。
- **打包/密度层的真实现状（2026-07-13 实测修正）**：分两条路径看。
  ① **pip API 路径**（`individual_parts[i].components[j].UV`，我们 notebook 的
  消费方式）：每个 chart 归一化到 unit square，chart 间尺度未定义（实测某 mesh
  离散达 318×）——密度完全不受控；
  ② **官方文件路径**（`save_results()` → `final_components.obj` → blender
  `pack_islands`）：导出合并 OBJ 时每个 chart 已按 **UV 面积 ∝ 3D 面积** 等比缩放
  （实测打包前后 chart 尺度 max/min=1.00），打包后面积加权 TD CVw 仅 0.05–0.08
  ——**事实上已达成 L1 均匀密度**（`pack_blender.py` 里被注释掉的
  `average_islands_scale()` 在此路径下是冗余的）。
  **真正留白的是密度的显式控制**：config 无任何 density/scale 参数、无 per-chart
  权重接口、无内容感知分配——即 L1 是隐式的固定行为，L2/L3（按内容/语义/用户
  指定分配预算）完全缺位。这正是本课题的接口位置。
- **许可注意（商用）**：主体 Apache-2.0，但内嵌 PartField 代码/权重是
  **NVIDIA non-commercial**；最优 packing 路径依赖付费 UVPackmaster。
  学术无碍；产品化需替换 PartField（自训分割或换开源分割）与 packer（自研，
  我们已有 MaxRects 雏形 pack2.py）。
- 仓库 9 个 issue 中**零个**涉及 texel density / chart scaling / packing——
  社区还没人往这个方向想。

## 4. 创新点的一句话表述（论文 pitch 草稿）

> Existing UV unwrapping minimizes distortion and chart count but treats the
> atlas pixel budget as uniformly distributed; existing content-aware
> parameterization requires the texture signal to already exist. We close this
> loop for AI texturing: given a white-model mesh and a reference image, we
> predict a texel-density demand field, and jointly decompose (part-based) and
> pack the UV atlas so that every part receives its perceptually-required pixel
> budget — before any texture is generated.

## 5. 竞争预警与时间窗

1. **Hunyuan3D Studio (腾讯, arXiv 2509.12815)** 的系统论文里已出现一句
   "context-aware UV segmentation that groups surfaces by material type and
   **texel density requirements**"——目前只是工程描述（无公式、无消融、无评测），
   但说明工业界已经闻到这个方向。**谁先把它形式化+评测谁得分**，建议档 A demo
   尽快跑通、尽早锁定叙事。
2. PartUV 作者组（Hillbot/UCSD）手握 part tree 与管线，做密度是他们自然的 next step
   （虽然 issue/论文毫无迹象）；SeamGPT/ArtUV 一系（腾讯）在 artist-style UV 上
   持续发力，也可能拐进来。
3. 缓解：我们的差异化护城河是**端到端生成验证**（同分辨率下只换布局、比生成纹理
   清晰度）——依赖组内 Meshy 生成管线积累，外部团队短期难以复制；
   两遍法（生成回路里实测内容密度）也天然长在生成管线上。
   TDF 若日后成熟可再叠加为第四路需求信号（当前已决策不作为依赖）。
