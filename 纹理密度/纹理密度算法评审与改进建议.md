# 纹理密度分配算法：评审结论与改进建议

## 1. 核心结论

当前方案的总体框架是合理的，可以概括为：

> 在唯一的物理量——纹理密度 \(\mathrm{TD}\)——下，通过 chart 级缩放实现均匀、内容感知和用户指定三种分配政策。

其中：

- **L1 均匀分配**：作为工业标准和验收底线；
- **L2 内容感知分配**：根据纹理内容需求重新分配有限 texel 预算；
- **L3 用户指定分配**：由美术或用户直接指定 part/chart 的预算；
- 三者改变的都是同一个物理量 \(\mathrm{TD}\)，不是三种不同“密度”。

当前方案已经具备以下优点：

1. \(\mathrm{TD}(f)=R\sqrt{A_{UV}(f)/A_{3D}(f)}\) 的定义正确；
2. 通过 chart 标量缩放兼容现有 UV 参数化与 packing 流程；
3. L1 使用面积加权 \(\mathrm{CV}_w\) 评价均匀度是合理的；
4. L2、L3 可以统一解释为对有限纹理面积预算的重新分配；
5. 绝对 TD 与图集分辨率之间保持线性关系，便于换算 px/cm。

当前最大的研究难点不在几何面积，而在：

> 如何定义一个稳定、可解释、与最终重建质量相关的内容需求信号 \(cw\)。

当前使用的“面内亮度标准差”可以作为 baseline，但不足以作为最终内容复杂度模型。

---

## 2. 建议明确区分三个量

建议在论文、文档和代码中显式区分：

| 变量 | 含义 | 类型 |
|---|---|---|
| \(cw(f)\) | 内容需求信号 | 非密度 |
| \(q_f\) | 相对线性 TD 倍率 | 无量纲 |
| \(w_f=q_f^2\) | 相对 texel 面积预算倍率 | 无量纲 |

定义：

\[
q_f=
\frac{\mathrm{TD}_{\mathrm{target}}(f)}
{\mathrm{TD}_{\mathrm{L1}}}
\]

由于纹理面积与线性密度的平方成正比：

\[
w_f=q_f^2
\]

因此 chart 的缩放系数应为：

\[
f_c
=
\sqrt{
\frac{\sum_{f\in c}A_{3D}(f)q_f^2}
{\sum_{f\in c}A_{UV}(f)}
}
\]

这与当前代码形式

\[
D_c=\sum_{f\in c}A_{3D}(f)w_f,
\qquad
f_c=\sqrt{\frac{D_c}{\sum_{f\in c}A_{UV}(f)}}
\]

完全一致，但需要明确：

> 当前代码中的 \(w\) 应解释为“纹理面积预算倍率”，不是“线性 TD 倍率”。

如果当前定义为：

\[
w(f)=\left(1+\alpha \widehat{cw}(f)\right)^\gamma
\]

则实际线性 TD 倍率是：

\[
q_f=\sqrt{w(f)}
=
\left(1+\alpha \widehat{cw}(f)\right)^{\gamma/2}
\]

如果希望 \(\gamma\) 直接作用于线性 TD，应改为：

\[
q_f=
\left(1+\alpha \widehat{cw}(f)\right)^\gamma
\]

\[
w_f=q_f^2=
\left(1+\alpha \widehat{cw}(f)\right)^{2\gamma}
\]

代码变量建议改为：

```text
cw                 # 内容需求信号
td_mult            # q，线性 TD 倍率
area_weight        # w=q^2，纹理面积预算倍率
chart_scale        # chart 的 UV 线性缩放量
```

## 3. 当前 L2 实际上是 chart-level 内容感知

当前 `layout_with_scales(w)` 对每个 chart 只应用一个标量，因此它只能改变 chart 之间的相对 TD，不能改变同一 chart 内不同 face 的相对 TD。

也就是说，逐面的 \(cw(f)\) 最终会被聚合成 chart 的平均预算：

\[
q_c
=
\sqrt{
\frac{
\sum_{f\in c}A_{3D}(f)q_f^2
}{
\sum_{f\in c}A_{3D}(f)
}
}
\]

所以当前系统严格实现的是：

> **chart-level content-aware texture density allocation**

而不是严格的：

> per-face adaptive texture density。

如果一个 chart 内同时包含纯色区域、文字、划痕和高频图案，单一 chart scale 会丢失大量逐面内容信息。

建议增加 chart 内部需求异质性指标：

\[
h_c=
\sqrt{
\frac{
\sum_{f\in c}A_f
\left(\log q_f-\log q_c\right)^2
}{
\sum_{f\in c}A_f
}
}
\]

其中 \(h_c\) 不是密度，而是衡量“一个 chart 是否适合用单一倍率表示”的评价指标。

当 \(h_c\) 过大时，可考虑：

1. 根据内容边界进一步切分 chart；
2. 在 chart 内使用非均匀参数化；
3. 接受当前 chart 只能获得平均预算。

## 4. 亮度标准差的局限

当前的内容信号：

\[
cw(f)=\operatorname{std}(\text{face 内亮度采样})
\]

主要测量的是亮度变化幅值，而不是纹理的空间频率或采样需求。

例如：

\[
Y_\nu(x)=a\sin(2\pi\nu x)
\]

在覆盖整数周期时：

\[
\operatorname{Var}(Y_\nu)=\frac{a^2}{2}
\]

与频率 \(\nu\) 无关。

因此：

- 低频缓慢变化的纹理；
- 高频快速变化的纹理；

可能得到相同的亮度标准差，但二者对 texel 数量的需求完全不同。

亮度标准差还存在以下问题：

### 4.1 对网格三角化不稳定

同一个连续纹理信号，在大三角形上可能得到较大标准差；将该三角形细分后，每个小三角形的标准差会下降。

因此，结果可能依赖 mesh tessellation，而不是只依赖真实纹理内容。

### 4.2 固定 24 个采样点缺乏尺度适应性

- 大 face 在源纹理中可能覆盖数千个 texel，24 点容易漏掉高频内容；
- 小 face 可能只覆盖少量 texel，24 点又可能冗余。

### 4.3 无法区分渐变、边缘和噪声

- 平滑渐变的标准差可能较大，但双线性插值很容易重建；
- 很窄的文字边缘面积很小，却可能非常重要；
- 高频随机噪声可能得分很高，但未必值得完整保留。

### 4.4 只看亮度会漏掉重要信号

亮度标准差无法充分描述：

- 等亮度色彩边界；
- normal map 高频变化；
- roughness 变化；
- opacity/mask 边缘；
- displacement 细节。

因此，建议将“亮度标准差”定位为：

> 一个简单的 contrast-based baseline，而不是最终的内容复杂度 oracle。

## 5. TD 是标量，但必须单独评价各向异性

纹理密度 \(\mathrm{TD}\) 仍然只有一个标量，但该标量不能完整描述两个切向方向上的采样率。

设三角形局部表面坐标为 \(x\)，UV 映射的 Jacobian 为：

\[
J_f=\frac{\partial u}{\partial x}
\]

像素空间映射为：

\[
K_f=R J_f
\]

设 \(K_f\) 的奇异值为：

\[
\sigma_1\geq\sigma_2>0
\]

则当前标量 TD 为：

\[
\mathrm{TD}(f)
=
\sqrt{\sigma_1\sigma_2}
\]

即两个主方向采样率的几何平均。

定义参数化各向异性：

\[
\kappa_f=\frac{\sigma_1}{\sigma_2}
\]

其中 \(\kappa_f\) 不是密度，而是评价指标。

例如：

\[
K=
\begin{pmatrix}
10&0\\
0&0.1
\end{pmatrix}
\]

有：

\[
\mathrm{TD}=\sqrt{10\times0.1}=1
\]

但两个主方向的采样率相差 \(100\) 倍。

因此：

> 相同的标量 TD 不代表两个方向上都具有足够采样率。

建议在现有 TD 指标之外，至少报告：

\[
P_{95,A}(\kappa_f)
\]

以及 chart 内 log-TD 偏差：

\[
e_f=
\log\frac{\mathrm{TD}(f)}{s_c}
=
\frac12\log\mathrm{stretch}(f)
\]

对应评价量：

\[
\operatorname{RMS}_A(e_f)
\]

\[
P_{95,A}(|e_f|)
\]

\[
\max |e_f|
\]

需要注意：当前 PartUV 的面积失真度量主要约束 Jacobian 的行列式，即面积变化，不能检测纯粹的方向拉伸。

## 6. 最合理的内容需求定义：Rate–Distortion

理论上最合适的 \(cw\) 不是某个局部滤波器响应，而是：

> 在一个区域增加一定 texel 预算后，最终重建误差能够下降多少。

因此，可以为每个 chart 测量一条 rate–distortion 曲线。

对每个 chart \(c\)，选择若干候选 TD 倍率：

\[
q_j\in
\{0.25,0.35,0.5,0.71,1,1.41,2,\ldots\}
\]

对每个 \(q_j\)：

1. 从高分辨率 reference 生成 chart 信号；
2. 按 \(q_j\) 对纹理降采样；
3. 使用与运行时一致的 bilinear、mipmap 等重建方式；
4. 在表面采样点或渲染视图上计算误差；
5. 得到误差曲线：

\[
E_c(q_j)
\]

最终求解：

\[
\min_{\{q_c\}}
\sum_c
\pi_c A_c E_c(q_c)
\]

满足：

\[
\sum_c B_c(q_c)\leq B
\]

其中：

- \(A_c\) 为 chart 的 3D 面积；
- \(\pi_c\) 为可选的重要度，如可见性、语义或用户权重；
- \(B_c(q_c)\) 为实际纹理预算；
- 理想情况下：

\[
B_c(q_c)\approx A_cq_c^2
\]

可以将内容需求定义为 L1 附近的边际收益：

\[
cw_c
=
-\left.
\frac{\partial E_c}
{\partial(q_c^2)}
\right|_{q_c=1}
\]

其含义是：

> 在当前均匀密度基准附近，额外增加单位纹理面积能够减少多少重建误差。

Rate–distortion oracle 的价值包括：

1. 自动区分空间频率、对比度、边缘、噪声和插值方式；
2. 直接给出固定纹理预算下的最优或近似最优分配；
3. 可作为低成本 \(cw\) proxy 的监督信号或评价标准；
4. 与当前“一 chart 一标量”的控制变量完全匹配。

## 7. 推荐的低成本内容信号

完整 rate–distortion 测试成本较高，因此生产中可以使用便宜的 proxy，但这些 proxy 应尽量逼近 rate–distortion oracle。

### 7.1 在表面世界坐标中计算信号变化

设源纹理为 \(I(u)\)，表面上的实际信号为：

\[
s(x)=I(u(x))
\]

表面梯度为：

\[
\nabla_Ss
=
J_u^\top\nabla_uI
\]

其中：

\[
J_u=\frac{\partial u}{\partial x}
\]

对于单个仿射三角形，表面 Hessian 为：

\[
H_Ss
=
J_u^\top H_uI J_u
\]

这样可以避免把源 UV 本身的缩放误认为纹理内容复杂度。

### 7.2 梯度适合表达方向，二阶信号适合表达重建需求

可以定义一阶信号张量：

\[
G_f
=
\frac1{A_f}
\int_f
J_s(x)^\top WJ_s(x)\,dA
\]

它的：

- 特征向量表示主要变化方向；
- 特征值表示两个方向上的变化强度；
- 特征值比表示信号各向异性。

但是一阶梯度会对线性渐变产生较高响应，而线性渐变通常很容易被双线性插值重建。

因此建议：

- 使用梯度张量估计变化方向；
- 使用 Hessian、Laplacian 或多尺度带通能量估计标量采样需求。

### 7.3 多尺度带通能量

可以定义：

\[
c_f
=
\sum_\ell
\beta_\ell
\frac1{A_f}
\int_f
\left\|
B_\ell s(x)
\right\|_W^2\,dA
\]

其中：

- \(B_\ell\) 为不同尺度的 Laplacian 或带通滤波器；
- \(\beta_\ell\) 为不同频段的权重。

可比较：

\[
\beta_\ell\propto\omega_\ell^2
\]

近似梯度能量，以及：

\[
\beta_\ell\propto\omega_\ell^4
\]

近似 Hessian 或曲率能量。

最终可通过 rate–distortion oracle 拟合最合适的频段权重。

## 8. 内容各向异性与标量 TD 的统一表达

可以将内容信号张量归一化为单位行列式：

\[
\overline G_f
=
\frac{G_f+\varepsilon I}
{\sqrt{\det(G_f+\varepsilon I)}}
\]

满足：

\[
\det\overline G_f=1
\]

再定义目标采样 metric：

\[
M_f^\star=q_f^2\overline G_f
\]

其中：

- \(q_f\) 决定唯一的标量 TD；
- \(\overline G_f\) 只描述方向和各向异性；
- \(\det M_f^\star=q_f^4\)。

因此可以保持术语一致：

> TD 仍然只有一个；各向异性只是描述 texel 应沿哪些方向分配的附加结构。

在当前只支持 chart 标量缩放的系统中：

- \(q_f\) 可以用于 chart 预算聚合；
- \(\overline G_f\) 可以用于判断是否需要切 chart；
- 可以比较参数化方向与纹理主要方向是否匹配。

后续扩展可考虑每 chart 一个 \(2\times2\) 仿射变换，或直接优化 UV Jacobian。

---

## 9. 内容信号的支持域不应由 face 尺寸决定

Face 是离散化单元，不一定是视觉内容单元。

建议在固定物理尺度的 surface patch 上计算内容信号：

\[
c(x;r_1,r_2,\ldots)
\]

其中 \(r_\ell\) 是世界空间中的多尺度邻域半径。

然后再把 patch 信号积分或平均回 face/chart。

这样可以提高：

- 对重三角化的稳定性；
- 对 skinny triangle 的稳定性；
- 对跨 face 边缘和纹理方向的连续性；
- 对 chart 内内容异质性的判断能力。

当前固定“每 face 24 点”可继续保留为快速 baseline，但建议增加：

1. 面积相关的自适应采样数；
2. 分层或分块采样；
3. 基于纹理 mip pyramid 的多尺度采样；
4. 面积加权的表面邻域平滑。

## 10. 不建议直接使用 Shannon entropy

“信息密度”容易被理解为熵，但当前任务实际包含不同概念：

- **采样复杂度**：避免重建模糊需要多少 texel；
- **压缩复杂度**：编码该纹理需要多少 bit；
- **感知重要度**：误差是否容易被人察觉；
- **语义重要度**：脸、文字、logo 等区域是否必须清晰。

例如：

- 规则棋盘的 Shannon entropy 可以较低，但采样率需求很高；
- 随机噪声熵很高，但未必值得完整保存。

当前目标是有限 atlas texel budget 下的重建质量，因此核心应是：

> 重建 rate–distortion，而不是 Shannon entropy。

只有当优化目标变为压缩文件大小时，熵才应成为主要指标之一。

## 11. 多纹理通道的处理

不同纹理通道应使用不同误差定义：

### Albedo / Base Color

可在线性 RGB 中计算误差，或对亮度与色度分别加权。

### Normal Map

解码并归一化后，使用角度误差：

\[
E_n=1-n_{\mathrm{ref}}\cdot n_{\mathrm{rec}}
\]

### Roughness

应尽量在实际 BRDF 使用的参数域中计算误差。

### Opacity / Mask / Text

边缘通常应获得更高权重，可加入 edge-aware 评价。

### Displacement

可使用：

- 位移误差；
- 法线误差；
- 轮廓误差；
- 最终渲染误差。

如果多个通道共享 UV 与分辨率，可以分别计算所需倍率 \(q_{f,k}\)，然后采用：

\[
q_f=\max_k q_{f,k}
\]

作为保守策略，或使用平滑 soft-max。

更高质量的方案是在代表性视角和光照下渲染，直接优化最终图像误差。

## 12. 建议改进 \(cw\) 的归一化

当前：

\[
\widehat{cw}
=
\frac{cw}{\operatorname{median}cw}
\]

存在两个问题：

1. 普通 median 会受到三角形数量分布影响；
2. 大量纯色 face 时 median 可能接近零。

建议使用面积加权 log-domain 标准化：

\[
z_f=
\frac{
\log(\varepsilon+c_f)
-
\operatorname{med}_A\log(\varepsilon+c)
}{
\operatorname{MAD}_A\log(\varepsilon+c)
}
\]

然后定义初始 TD 倍率：

\[
q_f^{\mathrm{raw}}
=
\exp\left(
\beta\,
\operatorname{clip}
(z_f,-z_{\max},z_{\max})
\right)
\]

为了保证 L1 和 L2 的理想纹理面积预算一致，进行显式预算归一化：

\[
q_f
=
\frac{
q_f^{\mathrm{raw}}
}{
\sqrt{
\frac{
\sum_fA_f(q_f^{\mathrm{raw}})^2
}{
\sum_fA_f
}
}
}
\]

再进行裁剪：

\[
q_{\min}\leq q_f\leq q_{\max}
\]

最后：

\[
w_f=q_f^2
\]

实际 atlas 占用还会受到以下因素影响：

- packing fill；
- chart padding；
- mip padding；
- chart 周长；
- block compression 对齐。

因此，公平比较时最好固定：

> 实际 atlas 分辨率、padding、mip 规则和最终占用预算。

也可以对全局倍率 \(\lambda\) 做二分：

\[
q_f\leftarrow\lambda q_f
\]

直到 packing 后实际成本达到目标预算。

## 13. 推荐的实验设计

主实验应关注：

\[
\text{重建或渲染误差}
\quad\text{vs.}\quad
\text{实际纹理预算}
\]

建议比较以下方法：

1. L1 uniform；
2. face luminance standard deviation；
3. surface gradient energy；
4. multiscale Hessian/Laplacian energy；
5. rate–distortion oracle；
6. 用户或语义指定权重。

### 合成测试纹理

建议包含：

- 常量；
- 线性渐变；
- 相同幅度、不同频率的正弦；
- 相同频率、不同幅度的正弦；
- 不同方向的条纹；
- checkerboard；
- 单一 sharp edge；
- 等亮度色彩边缘；
- 随机噪声；
- normal-map 高频细节。

### 稳定性测试

#### 重三角化不变性

同一表面改变三角形划分后，内容分配是否基本不变。

#### 源 UV 重参数化不变性

改变源 UV 的旋转、缩放或 chart 划分后，内容需求是否基本不变。

#### 模型尺度一致性

- 相对 L2 分配应对模型统一缩放保持一致；
- 绝对 px/cm 应正确恢复世界尺度。

### 推荐评价指标

| 目标 | 指标 |
|---|---|
| L1 均匀度 | \(\mathrm{CV}_w\)、log-TD p95 |
| L2 分配质量 | 固定预算下的重建误差 |
| 最终视觉质量 | 渲染图像误差 |
| chart 标量适用性 | \(h_c\) |
| 参数化面积稳定性 | log-stretch RMS/p95 |
| 参数化各向异性 | \(P_{95,A}(\kappa_f)\) |
| 工程成本 | fill、padding、mip、实际内存 |

不建议只用“TD 与 \(cw\) 的相关性”作为主要结果，因为相关性不能直接证明纹理预算带来了更高重建质量。

## 14. 推荐实施顺序

### 阶段 1：修正变量语义

明确：

\[
w=q^2
\]

并同步修正：

- \(\gamma\) 的解释；
- \(W_{\max}\) 的解释；
- L3 用户输入的预算含义；
- 代码变量名称。

### 阶段 2：加入 Jacobian SVD 评价

保持 TD 为唯一密度，同时新增：

\[
\kappa_f=\frac{\sigma_1}{\sigma_2}
\]

作为参数化各向异性评价指标。

同时将 PartUV 的“chart 内 \(\pm25\%\)”表述改为基于实测的面积加权 log-stretch 统计，除非算法确实提供逐面点态界。

### 阶段 3：建立 chart-level rate–distortion oracle

对少量代表性模型和候选倍率离线计算：

\[
E_c(q)
\]

作为评价所有 \(cw\) proxy 的金标准。

### 阶段 4：比较低成本 proxy

按以下顺序推进：

\[
\text{亮度标准差}
\rightarrow
\text{表面梯度}
\rightarrow
\text{多尺度 Hessian/Laplacian}
\rightarrow
\text{oracle 拟合模型}
\]

### 阶段 5：处理 chart 内异质性

使用 \(h_c\) 判断单 chart 标量是否足够。

当 \(h_c\) 较大时：

- 进一步切 chart；
- 使用内容感知参数化；
- 或提供人工覆盖接口。

---

## 15. 最终推荐表述

建议将方法概括为：

> 本方法在唯一的物理量纹理密度 \(\mathrm{TD}\) 下，提出一种兼容现有 UV chart、packing 和美术流程的纹理预算分配框架。L1、L2 和 L3 分别表示均匀、自动内容感知和用户指定三种分配政策。标量 \(q\) 控制线性纹理密度，\(q^2\) 控制 texel 面积预算；各向异性由独立的单位行列式张量或 Jacobian 奇异值描述，不被定义为另一种密度。内容感知分配的核心目标是在固定实际纹理预算下最小化表面信号或最终渲染结果的重建误差。

当前方案可以作为可靠的工程基础，但后续研究重点应从“亮度标准差”转向：

1. **rate–distortion 内容需求定义；**
2. **表面域、多尺度、方向感知的低成本 proxy；**
3. **chart 内内容异质性检测；**
4. **实际 packing 成本约束下的公平预算优化。**
