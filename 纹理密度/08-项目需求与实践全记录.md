## 一、需求（我们要做什么）

**Leader 原始需求**：输入一张图片 + 白模，在白模上输出正确的纹理，且纹理满足**纹理密度控制**。

**课题定位**：把 texel density 从"事后度量"升级为 **UV 布局阶段的一等公民优化目标**。
管线三段（展开 / 密度分配+打包 / 纹理生成）中，研究增量在中段；PartUV 是地基（已开源），
Meshy/UniTEX 生成管线是价值兑现场。

**已确认的三个关键决策**（06 文档决策记录）：

| 决策项 | 结论 |
|---|---|
| 产出形态 | 产品 demo 先行、沉淀成论文 |
| 密度控制层级 | **L1 均匀=验收底线（业界标准）、L2 内容感知=主攻创新点、L3 用户指定=顺带接口** |
| 需求信号来源 | **不依赖 TDF**；几何 + 语义先验为主，两遍法（粗生成→实测→重排→重生成）作上界与端到端故事 |

**术语纪律**（notebook 术语澄清块）：全程只有一个物理量 TD(f)；L1/L2/L3 是同一密度的三种
**分配政策**（世界地图类比：等积地图 / 人口 cartogram / 行政规划）；cw 是需求信号、
scale 是 chart 级汇总、CV/CVw 是评价指标——都不是"另一种密度"。

## 二、我们的做法（怎么做的）

**核心公式链**（notebook 公式卡，7 条，标注了在用/理论/前瞻状态）：

```
TD(f) = R·√(A_UV/A_3D)                                  ← 唯一物理量
cw(f) → z(面积加权 log z-score) → q=exp(β·clip(z)) 预算归一+裁剪 → w=q²
demand_c = Σ A_3D·w  →  chart 缩放 f_c=√(demand/A_UV)  →  pack_islands 摆放
```
**语义要点**：q 是线性 TD 倍率、w=q² 是面积预算倍率；预算归一 mean_A(q²)=1 保证
L2 与 L1 严格同预算（纯重分配）。β=0 退化为 L1。

**五步管线**（notebook `code/notebook/partuv_td_test.ipynb`，全部执行验证）：

| 步骤 | 内容 | 关键实现 |
|---|---|---|
| STEP 1 | 加载 mesh（多材质自动合并） | trimesh force="mesh"（合并多贴图为单图+重映射 UV） |
| STEP 2 | PartUV → UV map | pip `partuv` API + FaceMatcher 面对应 + Blender pack_islands |
| STEP 3 | 纹理映射进新 UV | 纹素级 rebake：新UV→重心坐标→3D→原面(对齐+角序)→原UV 双线性 |
| STEP 3.5 | 预算/分辨率锚定 | 锚A 预算守恒 R=√(被引用纹素/fill)；锚B 密度守恒(px/世界单位)；R>4096→触发 multi-atlas |
| STEP 4 | 内容频率 → 密度需求 w(f) | 亮度 std oracle（升级路线：表面梯度→多尺度Hessian→R-D oracle） |
| STEP 5 | 密度驱动重排 + 全套对比 | 同分解同打包器，仅换 chart 相对大小；图集/3D热力图/裁剪三组对比 |
| 附录 | Multi-atlas | part 分组 + 每图集独立分辨率 R_k∝√(D_k/fill_k)（密度的量化档位） |

**架构原则**：密度层定 chart **相对大小**，成熟打包器（pack_islands）管摆放——打包只做
旋转/平移/统一全局缩放，不破坏密度分配。全程不改 PartUV C++ 核心（档 B 才需要）。

**代表性实测结果**（车轮 objaverse_92ff65…，7012 面，同预算 1024²）：
信息量均匀度 CVw(TD²/w) 0.631→0.433；top-10% 内容面 TD ×1.29~1.34；
最高细节 chart 像素 ×1.66~2.18（打包随机性内波动）；corr(logTD², logw)≈0.91。

## 三、已发现的特殊情况与坑（按严重程度排序，全部实测）

### 🔴 曾是真 bug、已修复

1. **双面重合孪生面击穿最近邻质心匹配**（复杂资产 `8d1b6fc…` 28.9% 面为重合孪生）：
   coverage 虚报丢面 5.79%（实际 PartUV 只真丢 2 面）；更严重的是 blender_repack 往返对应
   会给孪生一层取到**另一层的 UV**（布局损坏）。修复：`FaceMatcher`（质心桶式一对一 +
   法线判别 + kd 兜底），notebook 三处对应逻辑已替换，coverage 0.861→1.0000，无孪生资产零回归。
2. **preprocess() 归一化坐标系**：processed mesh 被中心化+等比缩放，与原 mesh 世界坐标
   对不上（某资产 scale=0.0624）——第一版 rebake 映射成功率 0%。修复：包围盒相似变换对齐。
   连带要求：**绝对 TD（px/cm）计算必须换回世界尺度**。
3. **固定 RES=1024 隐式砍预算**：多贴图资产（车轮原始 5×1024²）被烘进单张 1024² =
   预算砍 5×。修复：预算/密度双锚定单元格（车轮实测两锚均指向 2048²）。

### 🟡 上游行为/资产特性，需管线显式处理（已处理）

4. **PartUV 静默丢面**：孤立单三角形/小碎片组件不进任何 chart（1389004d 丢 33、7b3232 丢 28，
   面积均 <2%）。处理：covered mask 贯穿全部统计，热力图灰显，>5% 面积告警建议清理。
5. **τ=1.25 不是硬保证**：实测个别 chart 面积失真到 1.66（搜索期用简化网格代理估计）。
   处理：密度层按实测 chart 尺度分配，不盲信 τ；实测 chart 内 log-stretch RMS≈0.05（≈±5%，
   远好于名义界，写论文用实测统计）。
6. **pip API 与官方文件路径行为不同**：`Component.UV` 各自归一 unit square（chart 间尺度
   离散 318×）；`save_results` 导出则已按 3D 面积等比（官方文件路径事实上隐式 L1，
   CVw 0.05–0.08）。留白的是**密度的显式控制（L2/L3）**——我们的接口位置。
7. **`final_parts.faces` 是 per-part 局部索引拼接**，不可当全局面索引；真正 chart 结构在
   `individual_parts[i].components`，全局对应用 FaceMatcher。
8. **平铺 UV（uv>1）**：实测 trimesh 合并对 uv∈[0,2] 的重复采样内容忠实（颜色误差 0.015）——
   cw/rebake 读合并图安全。
9. **多材质/纯色材质/分辨率悬殊**（32²~2048² 混合 + baseColorFactor 材质）：trimesh 合并
   全部正确处理；rebake 输出天然完成"多图合一 + 废弃区域回收"（clock：2 张图利用率 7.4% →
   单图 38.7%）。
10. **trim-sheet/共享贴图资产不适合重排**（clock 案例）：多面共享同一贴图区域是省内存的
    故意设计，rebake 会把共享内容按 chart 复制展开。生产需**资产类型门控**（检测共享/平铺
    → 跳过重排）；AI 生成场景（unique texturing）不受此限。
11. **chart 数与论文 teaser 不复现**（手雷 35 vs 论文 26，官方 demo 亲测同为 35）：公开
    代码+公开 PartField 权重与论文实验版本的系统性差异（非随机，3 次重跑一致）。
    含义：**论文 baseline 数字必须自己实测**，不能引用其图表。ABC 夹具 11 vs 论文 12、
    table 38 vs 32——整体复现质量足够好。

### 🟢 指标与方法论层面（评审采纳后已修）

12. **逐面 CV 是陷阱指标**：微小三角形把 CV 抬高（某资产逐面 2.18 vs 面积加权 0.078）。
    一律用面积加权 CVw。
13. **w 的语义**（外部评审 §2）：w 作用于面积预算，线性 TD 是 √w——文档/参数已改为
    q/w=q² 双变量表述，β 作用于 log 线性密度。
14. **median 归一不稳**：大片纯色时 median≈0（contrast 爆炸 4600 万）、或一半面饱和顶 cap。
    已换面积加权 log 域 z-score + 显式预算归一。
15. **corr 不能当主结果**：分配跟着信号走 ≠ 重建质量提高。主指标应为**固定预算下的
    重建/渲染误差**（待落地，见遗留）。

## 四、遗留风险与开放问题

| 风险/问题 | 状态与对策 |
|---|---|
| **内容信号太弱**（亮度 std 不感知频率、依赖三角化） | 已知局限（评审 §4）。路线：表面梯度 → 多尺度 Hessian → **chart 级 rate–distortion oracle**（= 两遍法严格化 = Sander IMT 血脉，论文须引） |
| **chart 内异质性**（单标量丢失面级信息）——**已实测证实是瓶颈** | 车轮（h_c 超标 15/125=12%）上 L2 显著提升（CVw 0.631→0.433）；**Converse 鞋（86/223=39%）上 L2 反而退化（0.749→0.809, corr 仅 0.52）**（2026-07-14 独立复现 codex 审计）。结论：chart-level L2 只在"细节按 part/chart 分离"的资产上有效；h_c 占比是可预判的开关；**档 B tree cut 从"建议"升级为"必须"**，86 个超标 chart 是现成实验对象 |
| **评测主指标未升级** | 当前 headline 是 CVw/gain/crop；论文级需"固定预算下重建误差曲线" + 合成纹理组 + 三个不变性测试（重三角化/源UV/尺度） |
| **真丢的面没有 salvage** | 真实丢面很少（修复孪生 bug 后多数资产 ≈0），但生产需兜底：给孤儿面独立小 chart 或投影补丁 |
| **大网格性能** | rebake/repack 是 Python 逐面循环，49k 面约 10+ 分钟量级；生产需向量化/C++ 化 |
| **许可合规** | PartUV 主体 Apache-2.0，但内嵌 PartField 为 **NVIDIA non-commercial**、multi-atlas 官方路径依赖付费 UVPackmaster——研究无碍，**产品化须替换**（06 文档 Q6，待与 leader 对齐） |
| **竞争风险** | Hunyuan3D Studio 论文已出现 "texel density requirements" 字样（仅工程描述）；PartUV 组（Hillbot/UCSD）可能做密度。护城河=端到端生成验证+两遍法 |
| **待 leader 对齐** | 06 文档 Q4–Q8：评测用哪条生成管线、数据集范围、图集规格、里程碑节奏 |
| **生成场景的 reference** | R-D oracle 需要参考信号；白模场景来自第一遍生成（两遍法循环），Phase 2 落地 |
