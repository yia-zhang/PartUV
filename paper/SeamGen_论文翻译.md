## 摘要

UV 接缝放置是 3D 内容创作中关键但极耗人力的一步:美术师需要在图块形状、接缝隐蔽性、以及与语义/几何特征的对齐之间取得平衡。现有自动方法主要基于**逐物体优化**,要么依赖手工设计的目标函数来避免失真,要么借助预训练模型的代理(proxy)注入语义信息。但这些策略与工业生产管线中实际使用的接缝并不总是一致,产出的布局常偏离美术师偏好的接缝模式与实际生产需求。

为此,本文提出 **SeamGen**,一个与艺术家偏好和生产需求对齐的 **UV 接缝生成式模型**。SeamGen 不依赖人工设计的目标与约束,而是用**流匹配(flow matching)生成模型**,从大量现有接缝布局语料中学习**逐边接缝标签的分布**。一个关键挑战是:流匹配模型常用的 Transformer 架构是为点云等序列化表示设计的,无法自然地利用网格拓扑。为实现"网格原生"的学习,作者设计了 **Mesh Transformer** 骨干网络:在网格边上的**局部图注意力**与跨顶点的**全局自注意力**之间交替,同时捕获细粒度几何线索与长程拓扑连贯性。为进一步提升推理时的可控性与质量,作者利用流模型**免训练的 inpainting(补全)能力**,实现局部接缝修复与约束引导的接缝生成。

大量实验表明:通过从专业接缝布局数据中学习先验,SeamGen 生成的 UV 布局比基于失真的方法和基于语义代理的方法更贴合美术师创作偏好,感知质量更优。

## 1 引言

UV 展开是 3D 资产生产的关键步骤,通过网格表面的 2D 参数化来施加纹理与材质细节,其质量在很大程度上取决于**接缝放置**——即摊平之前在表面何处切割。手工放置接缝繁琐且依赖专业经验:理想的接缝不仅取决于几何失真,还取决于**感知可见性、语义部件边界与美术师惯例**。为减少人工,自动 UV 展开方法试图自动生成接缝与参数化,通常把 UV 展开表述为几何驱动的优化问题。

现有方法大多基于逐物体优化、以几何失真(保角/保面积)为目标(LSCM、ABF、SLIM、OptCuts 等);较新的方法如 **PartUV** 额外引入语义部件分割作为结构化图块布局的**代理**。然而,手工设计的目标与语义代理并不总与工业生产需求对齐,导致当前方法的布局常无法以有利于实际纹理创作的方式放置接缝——例如避开视觉显著区域、在重要表面区域保持纹理连续。

以狗的模型为例(图 2):美术师通常**避免在脸部放置接缝**;而 xatlas、PartUV 等方法为降低失真常在该区域引入过多接缝,破坏纹理连续性、增加可见瑕疵的风险。其根源在于:**显式优化目标**与真实场景中复杂的接缝设计偏好之间存在鸿沟——后者混合了由视觉经验与生产需求塑造的实用、感知、语义与几何因素。因此逐物体优化存在先天局限。这促使作者采用**数据驱动**的表述:直接从美术师制作的样例中学习接缝放置。

![图 2:与美术师手工接缝的对比](2607.12379_source/figures_png/artist.png)

*图 2:(a) 美术师手工接缝避开显著区域(如脸部),偏好不显眼的边界;(b–c) 以失真为导向的 xatlas 与 PartUV 产出的接缝布局与美术师参考差异明显,接缝常出现在视觉上重要的面部区域;(d) SeamGen 更贴合美术师的接缝分布,保住脸部完整、沿更自然的边界切割。*

为此,本文提出 **SeamGen**,把表面接缝切割表述为**网格图上的条件生成问题**:学习以输入几何为条件的美术师接缝布局分布。接缝生成的难点在于:接缝边只占全部网格边的一小部分,却要在不规则的网格连接上同时尊重几何、语义、美术师惯例与拓扑。作者没有对所有边独立地判别接缝,而是采用**流匹配**作为生成框架,学习把随机的逐边噪声变换为网格图上连贯的二值接缝指示。

一个关键挑战是:典型 Transformer 面向序列建模,更适合点云一类的表示,不能自然地考虑网格拓扑;但接缝决策既依赖**全局上下文**(把切割协调成覆盖全形状的连贯布局),也依赖**局部几何线索**(如二面角、曲率,决定接缝应放在何处)。为实现网格原生学习,作者设计了 **Mesh Transformer** 骨干:在网格边上的局部图注意力与跨顶点的全局自注意力之间交替,兼顾细粒度几何线索与长程拓扑连贯。

为支持局部控制,作者进一步把学到的流模型用作**免训练的条件先验**:给定稀疏的用户约束(如"无缝区域"或"指定切割边"),SeamGen 在采样过程中钳制(clamp)指定边、重新生成其余接缝布局。同一机制也支持**重叠自修复**:初始布局若产生无效的 UV 重叠,固定可靠接缝、只对问题局部区域重采样。这使约束编辑与推理期修正**无需额外训练、无需单独的修复模型**。

**主要贡献**:

1. 把 UV 接缝切割表述为**网格边上的接缝生成**问题,提出 SeamGen——第一个从真实 UV 数据中学习美术师接缝布局先验的**流匹配框架**;
2. 设计网格原生的 **Mesh Transformer** 用于网格图上的接缝预测,并把学到的流模型复用为免训练条件先验,支持**约束引导补全**与**重叠自修复**;
3. 实验表明:通过从专业接缝数据学习先验,SeamGen 生成的 UV 布局比各基线更贴合美术师风格、感知质量更优。

## 2 相关工作

### 2.1 几何驱动的 UV 映射

构建 UV 图集需要决定**如何把表面切成图块**以及**如何把图块摊平到 2D**。一长串工作通过手工几何目标解决这两个问题:经典表面参数化包括固定边界方法(Tutte 嵌入)、自由边界与保角方法(LSCM、ABF/ABF++、谱方法、边界优先方法等)、以及失真最小化/保单射的优化方法(ARAP、MIPS、SLIM、OptCuts 等)。这些方法能产出低失真且有效的参数化,但主要优化 2D 映射的**几何质量**(保角、保面积、最小拉伸、单射性)。

由于切割方式强烈影响参数化域与 UV 图集结构,许多几何驱动方法也显式研究接缝放置与图块分解:自底向上方法在失真界内生长合并图块(D-Charts、Bounded-distortion 等),构成 xatlas 与 Blender Smart UV Project 等实用工具的基础;另一些方法从谱分析、几何特征线、或平衡接缝复杂度与映射失真的变分目标中推导切割(AutoCuts、OptCuts、变分接缝等)。但它们仍由**预定义的几何准则**驱动,而非美术师 UV 布局中体现的语义与感知偏好。

### 2.2 学习式表面参数化

近年的工作探索了表面参数化与 UV 生成的神经表述。神经参数化方法学习连续的表面到平面映射或失真感知表示(AtlasNet、Neural Surface Maps、FlexPara、Nuvo、FAM、DA Wand 等),但它们通常用自监督几何目标(失真、重建、单射性损失)优化,且普遍把 UV 生成表述为**逐点映射**问题——这与本文关注的问题不同:本文预测的是网格连接上的**离散接缝布局**,从而在摊平之前确定图块分解。

更近期的方法更接近数据驱动的 UV 分解:**PartUV** 通过按失真与特征相似度聚合面片,把语义部件线索引入图块分解;**Strips as Tokens** 提出带原生 UV 分割的自回归网格生成框架,但其 UV 分解是网格生成的副产物,而非对已有输入网格的预测。同期工作也在探索数据驱动的接缝生成:**SeamGPT** 与 **MeshTailor** 从数据学习接缝布局,**SeamCrafter** 进一步引入强化学习改进自回归接缝生成。与它们不同,本文把接缝切割表述为**网格边图上的生成**,用流匹配为给定输入网格**一次性联合生成完整接缝布局**(非自回归)。

## 3 方法

![图 3:SeamGen 流水线](2607.12379_source/figures_png/pipeline.png)

*图 3:SeamGen 流水线概览。给定输入网格,SeamGen 把 UV 接缝切割视为网格图上的生成问题:构建带几何与拓扑特征的网格图,用 Mesh Transformer 参数化流模型。每个 block 先对携带位置信息的顶点特征做全局自注意力,再用带边特征的图注意力传播局部连接线索、以拓扑感知的方式同时更新顶点与边特征。从随机的边噪声出发,模型生成连贯的逐边接缝掩码,转化为接缝切割后将网格展开为 UV 图集。*

给定输入网格,SeamGen 生成一个**逐边接缝布局**(网格边上的二值接缝掩码),随后据此把网格展开为 UV 图集。该任务被表述为网格图上的**条件接缝生成**:模型学习以输入网格为条件的美术师接缝布局分布。为此:用流匹配实例化接缝生成(3.1 节);用同时捕获全局与局部网格线索的 Mesh Transformer 参数化(3.2 节);并把学到的流模型作为免训练的条件生成先验,支持稀疏用户约束下的接缝补全与问题 UV 区域的自动修复(3.3 节)。

### 3.1 把接缝切割视为生成问题

高质量的 UV 接缝布局很少是优化少数几个显式目标的结果。实践中,美术师的接缝体现了几何、感知、语义与实用因素的复杂组合:接缝常沿锐利折痕放置、藏在视觉不显眼的区域、与语义部件边界对齐,并且要便于后续的纹理绘制与失真控制。这些偏好大多由经验驱动、难以形式化为一组固定的手工规则,因此**学习美术师先验**是更有效的方案。

据此,作者把 UV 接缝切割表述为网格图上的条件生成问题。每个网格用其**网格图** $\mathcal{M}=(\mathcal{V},\mathcal{E})$ 表示,其中 $\mathcal{V}\in\mathbb{R}^{N\times 3}$ 为顶点坐标、$\mathcal{E}$ 为边集。给定 $\mathcal{M}$,目标是学习模型分布 $p_\theta(e\mid\mathcal{M})$ 来逼近美术师数据分布 $q_{\mathrm{data}}(e\mid\mathcal{M})$,其中 $e$ 是网格边上的二值接缝掩码,$e_i=1$ 表示边 $i$ 被选为 UV 接缝。

生成过程用**流匹配**实现:从定义在网格边上的高斯噪声出发,模型学习一个条件流,把样本运送(transport)到以输入网格为条件的美术师接缝掩码。训练时,采样美术师接缝掩码 $e_1$ 与逐边高斯噪声 $e_0$,构造插值状态

$$e_t=(1-t)\,e_0+t\,e_1,\qquad t\in[0,1],$$

并训练网络从"带噪边状态 + 网格几何 + 连接关系 + 时间 $t$"预测对应的速度场。推理时,从噪声出发对学到的流做积分得到连续边状态,再离散化为二值接缝掩码。

这样,方法不依赖人工指定的接缝放置规则或手工失真目标,而是**直接从数据中学习美术师 UV 接缝的分布规律**,使生成的布局反映专业接缝设计中隐含的几何、语义与感知先验。

### 3.2 Mesh Transformer 骨干网络

学习美术师接缝布局的关键挑战在于:接缝放置**同时是局部的和全局的**。局部上,一条边是否该切,强烈受附近表面几何影响(锐利折痕、曲率变化、部件连接处、被遮挡区域);全局上,这些边级决策必须在整个网格上协调,形成连续的切割路径、把表面划分成形状良好的 UV 岛。为此,作者用一个同时在网格连接与长程表面依赖上推理的 Mesh Transformer 来参数化流的速度场。

如图 3 所示,骨干网络把**全局自注意力**与**拓扑感知的局部图注意力**耦合起来:

- **特征初始化**:每个顶点特征由其 3D 坐标的频率位置编码初始化。对每条网格边 $(i,j)\in\mathcal{E}$,由端点坐标构造几何边描述子——相对偏移 $p_j-p_i$、边长、归一化方向、中点位置——为图注意力流提供局部几何线索;将其与当前带噪接缝变量 $e_{t,ij}$ 拼接后投影到边特征空间,再叠加正弦时间步嵌入,使网络能以流轨迹上的当前时刻为条件。
- **每个 block 含两个互补的注意力模块**:**全局自注意力**在整个网格上更新顶点特征,让相距很远的表面区域直接交换信息,为跨形状协调接缝决策提供长程上下文;**局部图注意力**再沿网格连接更新顶点与边特征,把拓扑感知的几何信息注入边表示——对每个顶点,以待更新的顶点特征为 query,邻居顶点特征及其关联边特征构造 key/value 做注意力聚合,使每个顶点从一环邻域收集局部几何与拓扑证据;然后用两个端点的更新后特征与该边此前的表示来更新每条边的特征。
- **输出**:由于网格边是无向的,消息传递时对每条边双向处理、预测前对两个方向的边特征取平均;最终边特征经归一化后投影为每条边的标量输出,即流匹配所用的逐边速度场。

通过交替进行全局上下文聚合与局部拓扑感知更新,骨干网络同时捕获接缝布局的全局组织与决定接缝落点的局部证据。

### 3.3 条件接缝补全(Conditional Seam Inpainting)

在生产 UV 工作流中,美术师常在部分约束下细化初始展开:保护显著区域、强制指定某些接缝、或修改造成重叠的图块。SeamGen 用**条件接缝补全**支持这一过程,把学到的流模型复用为生成先验。

参照条件扩散采样(RePaint),在整个采样过程中**钳制已知边**、让模型预测未知边。设 $\mathcal{K}\subset\mathcal{E}$ 为已知边集合,目标接缝标签为 $e_1^{\mathcal{K}}\in\{0,1\}^{|\mathcal{K}|}$。采样时流程与无条件生成相同,但每一步都用对应的流插值覆写已知边的值:

$$e_{t,i}=t\,e_{1,i}^{\mathcal{K}}+(1-t)\,e_{0,i},\qquad i\in\mathcal{K},$$

其中 $e_0$ 为初始高斯噪声。更新后的 $e_t$ 送入 Mesh Transformer 进行下一步去噪。由于骨干网络通过全局自注意力与局部消息传递双通道传播信息,被钳制的边会作为其余可编辑边的**上下文约束**,实现连贯的条件接缝生成。

该机制支持两个实用场景:

1. **约束引导的接缝补全**:给定稀疏的用户约束(无缝区域、必须切割的边),SeamGen 以这些输入为条件重采样其余未约束的边。生成布局既满足指定约束,又与学到的美术师接缝先验保持一致——用户给出高层编辑意图,模型通过条件采样补全其余接缝布局,提供一种便捷的粗粒度可控性。
2. **自动局部自修复**:由于生成目标并未显式强制"无重叠",少数采样布局在参数化后可能产生自重叠的 UV 岛。此时固定布局中可靠的部分、只把问题岛周围的区域设为可编辑,通过条件补全在受影响区域局部重采样接缝、保持其余布局不变。这把预训练模型复用为**局部接缝先验**,无需单独的修复网络、额外训练或手工后处理目标。

## 4 实验

### 4.1 实现细节

- **数据**:从公开 3D 资产(Objaverse、Objaverse-XL、Sketchfab)收集美术师制作的 UV 布局,从 UV 岛边界提取接缝标签;
- **训练**:流匹配阶段训练 200 个 epoch,**8 块 NVIDIA A100**,AdamW,固定学习率 $1\times10^{-5}$,batch size 16;采用 $x$-prediction 参数化 + $v$ loss;训练时对每个网格做随机旋转增强;
- **推理**:连续接缝预测以阈值 $\tau=0.0$ 二值化;随后用 Blender Python API、以提取的接缝按 angle-based 展开方法展开网格;
- 网络结构细节、数据收集与预处理流程见补充材料。

### 4.2 与基线的比较

对比四个自动 UV 展开基线:Blender Smart UV Project、xatlas、OptCuts、**PartUV**。

**定性比较**(图 8):Blender 与 xatlas 常把表面过度切碎成大量小 UV 岛,密集的切缝破坏语义连贯性、穿过视觉显著区域;OptCuts 的图块更整合(常把每个连通表面区域映射为单个大岛),但岛边界扭曲、与语义部件对齐较差;**PartUV 通过语义部件先验缓解了过度碎片化,但其失真驱动的摊平过程仍可能破坏语义连续性,接缝偶尔出现在视觉重要区域**。相比之下,SeamGen 的接缝布局更连贯、碎片切割更少、UV 岛形状更好;生成的接缝常与自然几何边界(折痕、部件连接处、被遮挡区域)对齐,同时避开视觉显著区域,给下游纹理编辑带来更友好的 UV 分解。

![图 8:定性比较](2607.12379_source/figures_png/qualitative.png)

*图 8:代表性网格上的 UV 展开结果定性比较。从左到右依次为 Blender、xatlas、OptCuts、PartUV、SeamGen(接缝红色高亮)。SeamGen 产出更完整连贯的接缝布局,UV 图集更接近美术师参考。*

**UV 空间纹理生成检验**(图 4):作者进一步通过与 2D 生成先验的兼容性来评估 UV 布局的语义质量——用 **GPT-Image-2** 以位置图与法线图为条件直接在 UV 空间合成纹理。由于 2D 训练的模型天然在连通区域内生成连续纹理,合成质量可作为 UV 布局语义连贯性与连续性的代理指标。UV 图块高度碎片化的基线常破坏全局语义上下文,因孤立岛之间缺乏结构连续性而出现严重渗色与断裂图案;SeamGen 保持了连贯的语义区域,显著缓解这些瑕疵、产出高度连贯的纹理。

- **传统几何指标**:一个有意思的发现——**美术师制作的 Ground Truth 布局在所有方法中面积失真最差(2.4238)、角度失真倒数第二**,说明**失真最小化并不是专业接缝放置的首要目标**。SeamGen 虽不显式优化失真,角度失真仍与现有自动方法相当,面积失真也在合理范围内。运行时间上,SeamGen(2.87s)略慢于 Blender、xatlas、PartUV,但远快于 OptCuts(568.53s),在艺术家对齐的接缝质量与效率之间提供了实用的平衡。

## 5 结论、局限与未来工作

**结论**:本文提出 SeamGen,一个直接在网格图上学习美术师 UV 接缝布局分布的流匹配生成框架。Mesh Transformer 骨干在网格边上的局部消息传递与跨全体顶点的全局自注意力之间交替,捕获生产级接缝放置所需的细粒度几何线索与长程上下文;继承自流匹配的免训练 inpainting 机制,用单个预训练模型同时实现用户引导的接缝生成与重叠 UV 岛的迭代修复,无需启发式后处理或辅助训练目标。实验表明,SeamGen 生成的接缝布局在接缝结构与 UV 岛统计上紧密贴合真值分布,在定量指标、定性比较与用户研究上均优于基于失真和基于语义代理的基线。

**局限**(三点):

1. **评估难题**:目前仍没有直接且被广泛接受的指标来度量"与美术师接缝布局的分布级对齐";本文用 UV 布局统计、VLM 评测与用户研究作为互补证据,设计更好的客观指标是重要的未来方向;
2. **偶发无效图集**:SeamGen 偶尔会产生无效的 UV 图集,最坏情况下少数问题岛需要**局部回退到传统展开方法**;
3. **规模限制**:当前实现对超过约 **3 万面**的网格可能超出 GPU 显存;未来的层级式图设计有望提升对高分辨率资产的可扩展性。

## 快速要点总结(TL;DR)

1. **问题与立场**:接缝好坏不只由失真决定,还包括隐蔽性、语义与美术惯例;实测发现**美术师真值接缝的失真反而是所有方法里最差的**——"降失真"并非专业接缝的首要目标。因此干脆放弃手工目标,直接**学美术师的接缝分布**。
2. **核心思想**:把接缝切割建模为**网格边图上的条件生成**——流匹配把逐边高斯噪声运送成二值接缝掩码($e_t=(1-t)e_0+te_1$,网络预测速度场),一次性联合生成全部接缝(区别于 SeamGPT/MeshTailor/SeamCrafter 的自回归路线)。
3. **架构**:Mesh Transformer——每个 block 先做跨顶点**全局自注意力**(长程协调),再做带边特征的**局部图注意力**(一环邻域几何证据,更新顶点+边特征);边描述子含相对偏移/边长/方向/中点,叠加时间步嵌入;双向处理无向边后取平均,输出逐边速度。
4. **免训练可控性**:RePaint 式钳制已知边 → 用户约束补全(无缝区/指定切缝)+ 重叠自修复(岛内加缝或连邻域重采样),单样本重叠率 26.5% → 3.5%。
5. **效果**:200 网格验证集上,接缝边数(471.5 vs GT 476.6)与 UV 岛数(62.7 vs 67.8)最贴近美术师;Gemini-2.5-Flash 评分 3.56、20 人用户研究 4.37 双双第一(PartUV 分别为 3.02/2.91 居第二);失真不占优(面积失真 1.91),运行 2.87s。训练:8×A100、200 epochs;>3 万面会爆显存。
6. **与 PartUV/ArtUV 的关系**:三篇构成一条演进线——PartUV 用"语义部件 + 几何启发式 + 递归搜索"(非学习参数化,失真有硬约束);ArtUV 学"软件初始 UV → 美术师精修"的参数化残差(切缝交给 SeamGPT);SeamGen 则把**切缝本身**变成生成问题、彻底数据驱动。SeamGen 论文明确指出 PartUV 的失真驱动摊平仍会把缝切进视觉重要区域(如脸部),这与 PartUV 自己"接缝落在语义边界更隐蔽"的叙事形成有趣的张力;但 SeamGen 在失真指标上明显更差(1.91 vs PartUV 1.08),两者的取舍正好互补。
