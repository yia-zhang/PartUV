## 摘要

UV 展开(UV unwrapping)将 3D 表面以最小的失真摊平到 2D,复杂表面通常需要被分解为多个图块(chart)。尽管该问题已被广泛研究,现有 UV 展开方法在 AI 生成的网格上仍经常失效——这类网格往往充满噪声、表面凹凸不平且几何条件很差。这些方法常常产生高度碎片化的图块和次优的边界,引入瑕疵并妨碍下游任务。

本文提出 **PartUV**,一个**基于部件(part-based)的 UV 展开流水线**,在保持低失真的同时生成数量显著更少、且与部件对齐的图块。PartUV 构建在近期的学习式部件分解方法 **PartField** 之上,在一个**自顶向下的递归框架**中将高层语义部件分解与新颖的几何启发式规则相结合。它保证每个图块的失真低于用户指定的阈值,同时最小化图块总数。整个流水线集成并扩展了参数化与打包(packing)算法,对非流形和退化网格做了专门处理,并进行了大量并行化以保证效率。

在四个不同来源的数据集(人工建模、CAD、AI 生成、常用测试模型)上的评测表明:PartUV 在**图块数量**和**接缝长度**上优于现有工具和近期的神经方法,失真水平与基线相当,在困难网格上保持很高的成功率,并支持诸如**按部件多图集打包**等新应用。

## 1 引言

UV 展开把网格的 3D 表面投影到 2D 平面,为每个 3D 表面点分配对应的 2D UV 坐标。这一步是 3D 内容创作流水线的基础:它使得细致的表面信息——如材质属性(基础色、粗糙度、法线贴图)以及环境光遮蔽、位移等辅助贴图——能够在 2D 空间中高效存储和编辑。

UV 展开的核心组成是**表面参数化**(surface parameterization):在尽量保持角度、面积等几何性质的前提下摊平 3D 表面。对几何复杂的网格,把整个表面摊平为单个 2D 图块会引入巨大失真,因此通常需要**图块分割**(chart segmentation,或称接缝切割 seam cutting),沿精心放置的接缝把网格切分为多个图块,使每个图块能以更低失真被摊平。最后,**UV 打包**(UV packing)把得到的图块排布到单位正方形(UV 图集)内,以最大化纹理空间利用率。

尽管 UV 展开研究已久,现有方法通常只针对"表现良好"的网格(如专业 3D 美术师制作的模型)调优,在更复杂的 **AI 生成网格**上经常失败。这类网格通常从神经场等值面提取(如 Marching Cubes),表面凹凸不平、包含大量小三角形、几何质量差(存在离散组件或孔洞)。在这类数据上,现有方法可能超时,或返回极度碎片化的图集——单个图块只包含一个或几个三角形。这种极端碎片化妨碍纹理绘制与编辑,在图块边界引入纹理渗色(texture-bleed)及烘焙/渲染瑕疵,并让下游应用背负难以处理的海量图块。

近期一些方法(Nuvo、FlattenAnything、FlexPara 等)用**神经场表示 UV 映射**并针对每个形状从头优化,以缓解碎片化。它们虽能有效控制图块数量,但通常需要运行 30 分钟以上,且仍有明显失真。另一些方法(OptCuts、AutoCuts)联合优化接缝长度与失真,同样计算昂贵。此外,不少现有方法只基于**局部几何性质**的启发式来分割图块或寻找接缝,而不利用几何或语义"部件"的概念,导致图块边界不直观、次优,常把语义连贯的区域切碎到多个图块中,进一步给纹理创作、部件级编辑、语义感知渲染等下游任务带来麻烦。

本文提出 **PartUV**,一个基于部件的 3D 网格 UV 展开流水线:生成数量少且与部件对齐的图块,同时保持低失真,处理鲁棒高效——通常在几秒到几十秒内完成。PartUV 构建在近期的学习式方法 **PartField** 之上(它为输入网格生成层级式部件树),并提出多个新颖的几何启发式规则,把简单的局部部件进一步分解为可低失真摊平的图块。PartUV 将 PartField 的高层语义分解与细粒度几何切割相结合,采用**自顶向下的递归搜索**:在保证每个图块失真低于用户指定阈值的同时最小化图块数量。图块摊平使用成熟的表面参数化算法(如 ABF++),图集布局使用经过验证的打包算法。为保证速度与鲁棒性,流水线进行了大量并行化与加速,并专门处理非流形和退化情形。

显式引入语义先验带来若干关键收益:

1. **改善分解**:减少对局部几何的过度依赖,而后者往往导致过度分割和超长运行时间;
2. **保持部件完整性**:防止图块边界切穿有意义的区域(如电视屏幕表面、人脸),便于编辑与渲染;
3. **更好的打包策略**:语义分组天然支持把相关图块组织在一起——放进同一图集,或分布到多个图集,提升组织清晰度,便于定位和编辑相关图块;
4. **接缝更隐蔽**:沿语义边界的接缝往往落在视觉上不显眼的位置,提升贴图模型的观感。

作者在四个数据集(人工建模、AI 生成、CAD、常用 3D 模型如 Stanford Bunny、XYZ Dragon)上评测 PartUV,并与常用工具 xatlas、Blender、Open3D 及近期神经方法(Nuvo)比较。PartUV 将输入网格分解为显著更少的图块(接缝也更短),同时保持与基线相当的低角度/面积失真;显式部件先验不仅让图块边界更贴合部件,还支持按语义部件分图集打包等新应用。此外,PartUV 成功率高,能处理大型、复杂、非流形网格,单个网格通常几秒到几十秒完成。

**图 2(流水线)**:给定网格 $\mathcal{M}$,先用学习式方法 PartField 预测部件感知特征场;对该特征场做聚类,得到网格的**层级部件树** $\mathcal{T}$。然后从根节点开始递归遍历 $\mathcal{T}$:对每个访问的节点 $\mathcal{P}$,应用两种新颖的几何策略把对应的部件网格分割为多组候选图块集 $\mathscr{C}$;每组图块用 ABF 算法摊平并评估失真。若失真超过用户指定阈值 $\tau$,则递归遍历 $\mathcal{T}$ 的左右子节点;否则采纳该分割及对应的 UV 映射。

## 2 相关工作

### 2.1 网格参数化

撇开网格分解不谈,**网格参数化**——把 3D 网格映射到 2D 域并最小化各种失真度量(等距、保角、等积)——是 UV 展开的基本操作。许多方法基于网格连接性直接优化 UV 坐标,在固定或自由边界条件下求解线性或非线性系统,包括:重心嵌入(Tutte 嵌入)、拉普拉斯特征分解技术、失真最小化方法(如 MIPS、LSCM 最小二乘保角映射)等。有些方法采用 local/global 策略,把逐三角形变换(如旋转)与全局拼接结合(如 ARAP)。相比之下,**基于角度的摊平(Angle-Based Flattening, ABF)** 系列方法先优化三角形内角再转换为 UV,角度保持通常优于 LSCM,但由于非线性求解器代价更高。**本文采用 ABF(ABF++)作为基础展开算法**,并与多种网格分解策略集成。

### 2.2 面向 UV 展开的网格分割

网格分割是 UV 展开的关键步骤,目标是把复杂 3D 网格切分为更简单的面片(图块),使每块能以最小失真摊平。现有方法大致分四类:

- **自顶向下**:递归细分网格,常用谱分析,或沿特征线、高曲率区域切割;
- **自底向上聚合**:从小种子(如单个三角形)生长图块,依据平面性或失真界合并,在质量约束下最大化图块尺寸;
- **切缝优化**:寻找最优接缝网络,最小化接缝长度、降低摊平失真(OptCuts、AutoCuts、几何图像等);
- **聚类式**:按几何相似性对网格元素分组形成连贯片段。

常用工具 xatlas、Blender、Open3D 主要采用**仅依赖局部几何性质的自底向上聚合**策略。相比之下,PartUV 通过自顶向下的递归树搜索,把学习模块提供的高层语义部件先验与新颖的局部几何分解策略结合起来。

### 2.3 学习式参数化与分割

用神经网络表示 UV 映射因其可微性与可组合性受到关注。Neural Surface Maps 开创了跨表面集合学习映射的方向;一些方法用循环一致性和失真损失从多视角图像联合优化 UV 参数化与 3D 重建;Nuvo 直接从采样的 3D 点求解参数化;更近期的方法(FlattenAnything、FlexPara)设计神经网络模拟切割、UV 变形、展开等物理操作。但上述技术都依赖**逐形状优化**,计算昂贵,单个形状常需数十分钟。少数前馈模型能高效处理特定任务,如类内纹理迁移(AUV-Net)和低失真面片选择。

在语义部件先验方面:传统学习式部件分割受限于部件标注数据集的规模,只能处理封闭类别;近期方法把 2D 视觉先验(SAM、GLIP)提升到 3D 做开放世界部件分割,但依赖逐形状优化,运行慢且对噪声敏感。**PartField** 提出前馈模型学习部件感知特征场,实现快速的层级式 3D 部件分解。语义先验直观上有利于 UV 展开,但**朴素的集成会产生次优结果**;PartUV 用一种新颖的自顶向下策略把语义分割与几何摊平交错起来解决这一问题。

## 3 方法

### 3.1 概述

对 3D 网格 $\mathcal{M}=(V,F)$,PartUV 把网格面片分解为少量互不相交的**图块(chart)**:

$$F=\bigcup_{k=1}^{K}C_{k},\quad C_{i}\cap C_{j}=\varnothing\ (i\neq j), \tag{1}$$

其中每个图块 $C_k$ 是**连通**的面片子集。使用基于角度的摊平算法(ABF++)把每个图块摊平到 2D 平面,得到映射:

$$\phi_{k}:C_{k}\longrightarrow\mathbb{R}^{2}, \tag{2}$$

于是每个顶点 $v_i\in V$ 依其所属图块获得 2D UV 坐标 $\mathbf{u}_i=\phi_k(v_i)\in\mathbb{R}^2$。

核心挑战在于把网格分解为图块 $\{C_k\}$——本质上等价于为网格分割找到最优接缝。xatlas、Blender Smart UV 等主流方法依赖几何启发式(如基于面法线或失真度量的自底向上贪心图块扩张)生成图块,但通常产生过度碎片化的图块和不直观、次优的边界,常把语义连贯的区域切碎。

与之相反,PartUV 采用**由粗到细的两阶段分解策略**:

1. **高层**:用学习式方法 PartField(见 3.2 节)把网格面片划分为语义连贯的**部件(part)**:$F=\bigcup_{m=1}^{M}P_{m}$,每个部件几何相对简单(如圆柱状的四肢、球状的脚趾);
2. **细层**:引入两种基于几何的启发式(见 3.4 节),把每个部件进一步切分为少量图块:$P_{m}=\bigcup_{n=1}^{N_{m}}C_{m,n}$,保证每个图块能以最小失真摊平到 2D。

为此,PartUV 采用**自顶向下的递归分解搜索**(见 3.3 节):在每个图块失真不超过用户阈值 $\tau$ 的前提下,最小化生成图块的总数。由于穷举式分解搜索计算开销巨大,论文引入加速与并行化技术。3.5 节还描述了保证流水线完整鲁棒所需的预处理与后处理,包括非流形与多组件网格的处理以及 UV 打包。

### 3.2 预备知识:PartField

PartField 训练一个**前馈神经网络**:输入 3D 网格,预测一个连续的、以三平面(triplane)编码的**部件特征场**。它通过在部件标注 3D 数据以及带 2D 伪部件标签的大规模无标注 3D 数据上的大规模对比学习,学到了语义与几何部件的通用层级概念。对任意 3D 点,插值三平面表示即可得到其高维部件特征;特征相似(以余弦相似度衡量)的点更可能属于同一部件。

### 3.3 自顶向下递归树搜索

用 PartField 得到层级部件特征场后,先为每个三角面片计算代表性部件特征:在面内均匀采样 $s$ 个 3D 点并平均其点特征。随后对这些面特征做**凝聚层次聚类**(agglomerative clustering),构建**层级部件树** $\mathcal{T}$:叶节点是单个三角面片,根节点是整个网格。

一种直接的做法是用 PartField 直接生成固定数量的部件、再逐个摊平;或者自根向下自适应遍历 $\mathcal{T}$,检查每个节点的几何能否低失真摊平为单个图块。但作者观察到,**仅依赖 PartField 生成的部件往往结果次优**:PartField 主要关注语义或粗粒度几何划分,不擅长与 UV 相关的细尺度分解(例如把圆柱形或球形区域准确切分为失真最小的图块)。因此,PartUV 让 PartField 主要负责**高层语义分解**(得到几何更简单的子部件),再用两种几何启发式(3.4 节)把语义部件进一步切成便于低失真摊平的小图块,**搜索算法把这两种策略交错执行**。

形式化地,PartUV 采用算法 1 的自顶向下递归分解搜索,在图块数量与失真约束之间取得最优平衡。给定子树节点 $\mathcal{P}$(初始为 $\mathcal{T}$ 的根)、失真阈值 $\tau$ 和图块预算 $B$(初始为 $\infty$),算法为 $\mathcal{P}$ 的网格寻找满足三个条件的有效图块分解:每个图块失真 $\le\tau$;图块在 2D 中互不重叠;图块总数 $\le B$。若不存在可行分解,返回失败($\bot$)。

**算法 1:PartTreeSearch(部件树搜索)**

```text
输入:PartField 子树节点 P,失真阈值 τ,图块预算 B
输出:P 对应网格的图块集 𝒞 = {C₁, …, C_k},满足每个图块失真 ≤ τ 且 k ≤ B;
      若不存在这样的集合则返回 ⊥

procedure PartTreeSearch(P, τ, B):
    if B < 1:                                   ▷ 预算耗尽
        return ⊥

    H₁ ← GenCandidatesH1(P.mesh, 10)            ▷ 启发式 1(Normal),生成 1..10 块的候选分解
    for 每个候选图块集 𝒞 ∈ H₁:
        ParameterizeABF(𝒞)                      ▷ ABF 摊平并计算失真

    if min_{𝒞∈H₁} 𝒞.dist > τ:                   ▷ H1 无可行候选
        L ← PartTreeSearch(P.left_child,  τ, ∞)
        R ← PartTreeSearch(P.right_child, τ, ∞)
        return L ⊕ R                            ▷ 合并左右子树的图块集
    else:
        H₂ ← { GenCandidateH2(P.mesh, τ) }      ▷ 启发式 2(Merge)
        ParameterizeABF(H₂[0])
        S ← { 𝒞 ∈ H₁ ∪ H₂ | 𝒞.dist ≤ τ 且 NoOverlap(𝒞) }
        𝒞_best ← argmin_{𝒞∈S} 𝒞.count           ▷ 取图块数最少的可行解

        ▷ 继续递归,检查是否存在更优解
        B′ ← min(B, 𝒞_best.count − 1)
        L ← PartTreeSearch(P.left,  τ, B′ − 1)
        R ← PartTreeSearch(P.right, τ, B′ − L.count)
        𝒞_comb ← L ⊕ R
        is_valid ← 𝒞_comb.dist ≤ τ 且 NoOverlap(𝒞_comb)
        if 𝒞_comb.count < 𝒞_best.count 且 is_valid:
            𝒞_best ← 𝒞_comb
        return 𝒞_best
```

算法流程解读:

- 首先用**主几何启发式(Heuristic1,即 Normal)**为节点 $\mathcal{P}$ 的网格生成候选图块分解。每个候选图块集 $\mathscr{C}=\{C_i\}$ 至多含 $t$ 个图块,逐一用 ABF 摊平并评估失真。
- 若 Heuristic1 的所有候选(至多 $t$ 块)都不满足失真约束($\delta_{\min}>\tau$),说明该部件仍太复杂,则利用 PartField 树切分网格,**不设预算**地递归搜索左右子树,再合并两侧的最优图块集。
- 反之,若 Heuristic1 找到可行分解,则再用**次级启发式(Heuristic2,即 Merge)**尝试得到更少的图块;在所有满足失真与无重叠约束的候选中,选出图块数最少的最优解 $\mathscr{C}_{\text{best}}$。
- 在接受该解之前,算法做最后一次检查:以**收紧的预算** $B'=\min(B,\ \mathscr{C}_{\text{best}}.\text{count}-1)$ 递归探索左右子节点(左子树预算 $B'-1$,右子树预算 $B'-L.\text{count}$)。若子树搜索的组合解有效且图块更少,则替换当前最优解。
- 预算 $B$ 防止搜索过深:随搜索推进 $B$ 不断收紧,一旦候选超出预算递归即停止。实践中搜索很少走得很深。

通过把 PartField 的层级语义引导、细粒度几何启发式与系统的递归探索策略性地交错,该分解搜索得到语义连贯、失真有界且非常紧凑(图块数很少)的网格参数化。

### 3.4 基于几何的部件分解

本节介绍两种启发式——**Normal(法线)**与 **Merge(合并)**——用于进一步分解 PartField 生成的、几何简单的部件 $P_m$,目标是 $P_{m}=\bigcup_{n=1}^{N_{m}}C_{m,n}$,使每个图块都能低失真摊平。

**Normal 启发式**(对应算法 1 中的 `GenCandidatesH1`,发表版第 5 行):基于面法线。对 $P_m$ 网格的面法线做凝聚层次聚类,把三角面片划分为若干图块,每个图块由法线相近的**连通**面片组成。对每个 $P_m$ 该聚类只执行一次,产生 $t$ 个候选分解(分别含 $1$ 到 $t$ 个图块;实验取 $t=10$)。对每个候选分解,用 ABF 摊平并评估失真。由于 ABF 以保角为目标,失真用**面积拉伸(area stretch)度量**量化:

$$\text{distortion}(\mathscr{C})=\max_{C\in\mathscr{C}}\left(\frac{1}{|C|}\sum_{f\in C}\max\left(\text{stretch}(f),\ \frac{1}{\text{stretch}(f)}\right)\right), \tag{3}$$

其中 $C$ 为由多个连通面片组成的单个图块,$\mathscr{C}$ 为分解中所有图块的集合。三角面片 $f$ 的拉伸定义为:

$$\text{stretch}(f)=\frac{\text{area}_{2D}(f)}{\text{area}_{3D}(f)}\Bigg/\left(\frac{\sum_{f'\in C}\text{area}_{2D}(f')}{\sum_{f'\in C}\text{area}_{3D}(f')}\right), \tag{4}$$

即单个面片的 2D/3D 面积比,相对于其所在图块整体 2D/3D 面积比的归一化值。

值得注意的是,PartField 与 Normal 启发式都使用凝聚层次聚类把面片分组为部件/图块,**区别在于所用特征**:PartField 用学到的高层部件特征,Normal 用低层几何面法线。两者思路一致、实践互补。

**Merge 启发式**(对应算法 1 中的 `GenCandidateH2`,发表版第 14 行):Normal 简单快速、效果通常令人满意,但作者还提出了计算更昂贵、可能产生更少图块的第二种启发式 Merge。给定部件 $P_m$:

1. 计算其**有向包围盒(OBB)**;
2. 给每个三角面片分配 1~6 的标签,对应与其法线最接近的 OBB 面法线;
3. 结合面片连通性与标签,把面片分割为多个连通组件;
4. 组件按大小从小到大排序,从共享边最长的邻居开始,迭代地尝试把每个组件与相邻组件合并:每次合并尝试都临时合并两个组件并用 ABF 摊平合并后的图块,**若满足失真阈值且无重叠则接受合并**;
5. 直到无法再进行有效合并,返回最终图块集。

由于 Merge 常从大量小组件出发,且迭代合并过程要多次调用 ABF,其代价显著高于 Normal,但可能得到图块更少的更优分解。因此,**仅当 Normal 返回可行(admissible)分解时才调用 Merge**。

### 3.5 运行时优化与前后处理

**运行时优化**。为对抗递归分解与反复调用 ABF 的计算代价,分解过程采用两个关键策略:

1. **并行化**:自顶向下搜索中对左右子树的所有递归调用全部并行,充分利用多核处理器,显著加速整体分解;
2. **失真代理(surrogate distortion)**:为避免在稠密网格上反复调用 ABF 的高开销,使用 GPU 加速的网格简化算法(PAMO,Oh et al. 2025)生成候选图块的低分辨率近似;简化时**固定图块边界**,以保留与 UV 映射相关的几何结构。中间评估阶段用这些简化图块快速估计失真;最终图块集确定后,再对原始高分辨率网格运行 ABF,得到精确的 UV 坐标与失真度量。

**非流形与多组件网格**。ABF 假设输入图块是流形且连通的表面,因此需要额外处理:

- **非流形网格**:检测所有非流形边(被超过两个面共享的边),通过复制共享顶点、把每条这样的边拆分为 $N-1$ 条独立边($N$ 为关联面数)来解决,将结构转化为适合摊平的流形形式;
- **多连通组件网格**:先照常进行 PartField 引导的层级分解;若某部件网格包含多个不连通组件,则在该层跳过启发式分解,直接递归探索 PartField 层级的左右子树;若到达预设递归深度后 PartField 仍无法把多个组件分开,则回退为对该层的每个连通组件单独应用几何启发式与 ABF 摊平,以避免过度碎片化的分解。

**UV 打包**。本文重点在于把 3D 网格分解为图块并生成低失真 2D 参数化,方法与各种现有 UV 打包算法完全兼容。其独特之处在于:图块分解按**语义部件**分组,支持更结构化、面向应用的打包策略——例如同一部件的图块在打包时聚在一起;或把部件组打包到任意数量的 UV 图集(多个 $[0,1]^2$ 空间),各图集间语义均衡分布。这种语义层级不仅提升组织清晰度,还有利于纹理绘制/编辑等下游应用:同一部件的图块在空间上彼此邻近,便于整体操作。

## 4 实验

### 4.1 实现细节与评测设置

**实现细节**。PartField 在 Objaverse 上用 8 块 NVIDIA H100 训练 15 天。推理时每个面采样 10 个点、平均其特征,用于面聚类和部件树构建。PartUV 核心流水线用 **C++17** 实现,最终按组打包使用 **UVPackmaster**。所有实验的失真阈值取 **$\tau=1.25$**。ABF++ 每次调用运行 5 次迭代,并沿用 Blender 的梯度提前停止条件。网格简化由曲率相关阈值 $10^{-4}$ 与最大迭代次数 1000 控制。所有参数在全部实验中固定。所有方法在配备 96 核 Intel Xeon Platinum 8468 CPU 和一块 NVIDIA H100 GPU 的集群节点上评测。

**评测数据集**(四个,覆盖不同来源、质量与风格):

| 数据集 | 规模 | 说明 |
|---|---|---|
| Common Shapes | 24 | 图形学常用测试模型(Bimba、Igea、Stanford Bunny 等),GitHub 仓库提供处理好的 .obj |
| PartObjaverseTiny | 200 | Objaverse 子集,高质量人工建模网格,多组件、表面光滑 |
| ABC | 100 | CAD 机械模型数据集(取初始批次前 100 个),锐利与光滑特征并存 |
| Trellis | 114 | 近期 3D 扩散生成模型产出的 AI 生成网格,噪声大、几何质量低,挑战最大 |

**评测指标**(四个角度):

1. **图块数量**:逐形状统计图块数,报告数据集上的平均值与中位数;
2. **接缝长度**:所有图块边界边长度之和(UV 坐标归一化到 $[0,1]$),报告中位数;
3. **角度(保角)失真**:计算每个面切向量与副切向量的余弦,形状失真定义为 1 减去所有面的平均余弦(沿用 Nuvo 的定义;**越接近 1 越好**,表中 ↑),报告数据集平均;
4. **面积(等积)失真**:分两种——**area distortion(面积失真)**按公式 (3) 定义、取每个形状失真最大的图块;**overall area distortion(整体面积失真)**把所有图块的所有三角形汇总、逐个计算失真再平均。两者都先逐形状计算再对数据集平均;三角形级失真值截断到最大 10。整体面积失真可能被面数"平滑",而面积失真更能突出 UV 映射中的问题区域。未报告 stretch $L_2$/$L_\infty$,因为它们是逐三角形指标,在三角形翻转或面积接近零时会发散到无穷——这在 xatlas 等基线中很常见,直接比较意义不大。

### 4.3 应用与分析

- **纹理编辑与替换**:UV 映射碎片化更少,纹理图能更有效地在 2D 空间中编辑。实验展示了在 UV 图上干净地放置会议 logo(Blender/xatlas 的碎片化布局做不到),以及把纹理替换为各种平铺图案——xatlas/Blender 产生的大量只含几个三角形的小图块会在网格表面留下明显瑕疵,PartUV 的视觉质量显著更好。
- **纹理压缩**:图块之间总需要 padding;布局越碎,所需 padding 越多,渗色(color bleeding)风险越大。把 UV 贴图分辨率从 1024 降到 128(手游常见设置)时,xatlas/Blender 的纹理出现明显渗色,PartUV 则没有此问题。
- **多图集打包(Multi-Atlas Wrapping)**:PartUV 支持基于部件的 UV 打包——给定期望的图集数量,自动提取语义部件并跨多个图集打包,便于纹理编辑等下游应用。
- **自适应阈值调节**:用户可指定失真阈值 $\tau$,灵活控制图块数量与失真之间的权衡。

## 5 失败案例讨论

该方法在输入网格**拓扑很差**时会遇到困难。例如,它无法处理包含**自相交**的 3D 网格——此时算法可能为了解决相交而深度递归,导致图块碎片化。此外,当输入网格本身**极度碎片化**(如超过 1000 个组件)时,展开结果也会高度碎片化;这类情况可能需要先重网格化(remeshing)再做 UV 展开。不过,该方法在组件较少的一般网格上表现良好,且**不要求网格水密(watertight)或流形**。

## 6 结论

本文提出 PartUV,一个新颖的 UV 展开框架,策略性地把学习式方法提供的语义部件先验与两种新颖的几何启发式相结合。PartUV 在图块数量上显著优于现有方法,同时保持低失真,且图块边界与语义部件对齐。论文通过多个应用展示了该流水线的优势。

- **A2.3 只用 Merge(去掉 Normal)**:Merge 从 OBB 投影出发,在凹凸不平的网格上初始就可能得到数百个组件,全部尝试合并开销巨大(如 bimba 网格运行时间从 147 秒涨到 892 秒以上),且合并常产生重叠、反而增加图块数。Merge 只在较简单形状上更优,最适合作为 Normal 的补充、仅在 Normal 已给出足够好的结果时调用。

## 快速要点总结(TL;DR)

1. **问题**:传统 UV 展开(xatlas/Blender/Open3D)只用局部几何启发式,在 AI 生成的低质量网格上产生成百上千个碎片图块;神经优化方法(Nuvo 等)图块少但要跑 30 分钟以上且失真大。
2. **核心思想**:用学习式部件分解(PartField)提供**高层语义先验**,构建层级部件树;用两种几何启发式(**Normal**:法线凝聚层次聚类;**Merge**:OBB 六向标签 + 迭代合并)做**细粒度低失真切分**;二者通过**带预算的自顶向下递归树搜索**(算法 1)交错执行,在"每个图块失真 ≤ τ"约束下最小化图块总数。
3. **工程保障**:递归全并行;GPU 网格简化做失真代理加速中间评估;非流形边拆分、多组件回退处理;兼容任意打包器并支持按部件分组/多图集打包。
4. **效果**:四个数据集上图块数比 Blender 少一个数量级以上(Common Shapes 上 1/31)、接缝更短、失真与最优基线相当、成功率 100%,单网格几秒到几十秒;支持纹理编辑、低分辨率抗渗色、多图集打包等应用。
